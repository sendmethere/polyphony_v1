# Polyphony — 개발 명세 (Build Spec)

> 이 문서는 그대로 개발 착수에 사용한다. 기획 맥락은 [plan.md](./plan.md), 의사결정 로그는 [log.md](./log.md).
> 작성일 2026-06-22.

## 개요 & 목표
대화주의(dialogism) 철학을 구현한 실시간 토론 도구. **질문 → 의견 수집 → AI 군집화 → 세분/통합 후속질문 제안 → 재수집**의 사이클을 가로 스크롤 캔버스로 시각화한다. 절대 원칙: **무대(학생·교사 화면)에서 어떤 의견도 우세하게 보이지 않는다.** 목표는 정답 수렴이 아니라 다양한 voice와 그 전개의 가시화.

## 스코프 (MVP)
**포함:** 세션 생성·참가, 3뷰(학생/관리자/교사), 핵심 라운드 사이클, 가로 열(column) 캔버스, 우세 은폐 규칙, Firebase 실시간 동기화, API 키 설정, 수동 종료+읽기전용 회고.
**제외(2순위 이후):** 분기(계보) 뷰, 폴리포니 미터, AI 회고 요약, JSON 내보내기, 학생 반응 모드, 자동 종료, 군집 수동 편집, 인증/계정.

## 기능별 상세 명세

### FE-1. 세션 생성 & 설정 (관리자)
- **설명:** 관리자가 새 세션을 만들고 API 키·모델을 설정한다.
- **사용자 스토리:** "진행자로서 세션을 만들고 내 API 키를 넣고 싶다, 그래야 AI 군집화를 쓸 수 있다."
- **수용 기준:**
  - [ ] 세션 생성 시 고유 4~6자 코드가 발급되고 Firestore에 `Session` 문서가 생성된다.
  - [ ] 설정 화면에서 Anthropic API 키·모델을 입력하면 **관리자 브라우저 localStorage에만** 저장된다(Firestore에 키를 쓰지 않는다).
  - [ ] 키 미설정 상태에서 "AI 군집화" 시도 시 안내 메시지가 뜬다.

### FE-2. 세션 참가 (학생/진행자)
- **설명:** 코드 + 역할 선택 + 닉네임으로 입장.
- **사용자 스토리:** "학생으로서 코드와 닉네임만으로 빠르게 참여하고 싶다."
- **수용 기준:**
  - [ ] 코드 입력 → 역할(학생/진행자) 선택 → 닉네임 입력으로 입장. 로그인 없음.
  - [ ] 입장 시 `Participant` 문서가 생성된다.
  - [ ] 잘못된/종료된 코드는 거부된다.

### FE-3. 질문 발행 & 응답 수집
- **설명:** 관리자가 질문을 발행하면 학생 화면에 즉시 뜨고, 학생이 응답을 제출한다.
- **사용자 스토리:** "진행자로서 질문을 던지면 모든 학생 화면에 바로 떠야 한다 / 학생으로서 내 생각을 적어 제출하고 싶다."
- **수용 기준:**
  - [ ] 발행 시 `Round`(status=`collecting`)가 생성되고 학생 화면이 실시간 갱신된다.
  - [ ] 학생은 현재 질문 + 입력칸만 본다(기본 모드). 다른 의견·군집·우세도는 안 보인다.
  - [ ] 제출 시 `Opinion`이 저장되고, 학생은 "다음을 기다리는 중" 상태가 된다(재제출/수정 정책은 가정: 1인 1응답, 수정 가능).
  - [ ] 관리자가 "수집 마감"을 누르면 Round status=`closed`.

### FE-4. AI 군집화 (관리자 기기)
- **설명:** 수집된 의견을 Claude가 의미 기반으로 군집화하여 군집안을 제시.
- **사용자 스토리:** "진행자로서 흩어진 의견이 어떤 관점들로 나뉘는지 AI 도움으로 보고 싶다, 단 AI가 결론을 강요하면 안 된다."
- **수용 기준:**
  - [ ] "AI 군집화" 실행 시 관리자 기기에서 Anthropic API를 호출한다(학생/교사 기기는 호출하지 않는다).
  - [ ] 군집은 라벨 + 요약 + 소속 의견으로 제시되며, **외톨이(어디에도 안 묶이는) 의견은 별도로 보존**된다(임의 흡수 금지).
  - [ ] 군집은 "정답 후보"가 아니라 "서로 다른 관점"으로 표현된다(프롬프트가 우세/순위 매기기를 금지).

### FE-5. 군집 확정 & 후속질문 선택 (관리자)
- **설명:** AI 군집안과 세분/통합 후속질문 후보 중 관리자가 선택·수정해 다음 라운드를 발행.
- **사용자 스토리:** "진행자로서 AI 제안을 그대로 쓰거나 손봐서 다음 질문을 내보내고 싶다."
- **수용 기준:**
  - [ ] AI가 후속질문 후보를 **세분(diverge)/통합(converge) 방향별로 2~3개** 제시한다.
  - [ ] 관리자가 군집 라벨·구성과 질문을 편집할 수 있다.
  - [ ] 확정·발행 시 새 `Round`가 생기고 `Cluster.parentClusterIds`로 이전 라운드 군집과의 계보가 연결된다.

### FE-6. 가로 스크롤 캔버스 — 열 뷰 (모든 뷰 공통, 표현만 다름)
- **설명:** 라운드를 세로 열로, 시간을 좌→우로 펼친다. 각 열 = 질문 + 그 라운드 군집들.
- **사용자 스토리:** "토론이 어떻게 전개됐는지 가로로 스크롤하며 한눈에 보고 싶다."
- **수용 기준:**
  - [ ] 라운드가 늘면 오른쪽에 열이 추가되고 자동으로 최신 열로 스크롤된다.
  - [ ] **학생·교사 무대:** 군집 카드의 크기·색 채도·위치가 모두 동등. 인원수·카운트 미표시.
  - [ ] **관리자 뷰:** 동일 캔버스에 군집별 인원수 등 내부 지표가 함께 표시된다.
  - [ ] 열 사이 연결선이 이전→현재 군집의 세분/통합 관계를 나타낸다.

### FE-7. 모니터링 뷰 (교사)
- **설명:** 조작 버튼 없는 읽기 전용 관찰 대시보드.
- **사용자 스토리:** "관찰자로서 다양성이 늘고 있는지, 누가 침묵하는지를 차분히 보고 싶다."
- **수용 기준:**
  - [ ] 실시간 참여 현황(제출/미제출 수)을 본다.
  - [ ] 군집별 인원·분포, 개별 응답(닉 기준 추적, 무대엔 익명)을 본다.
  - [ ] (Should) 폴리포니 미터로 발산↔수렴 추이를 본다.
  - [ ] 진행 조작 UI가 노출되지 않는다.

### FE-8. 세션 종료 & 회고
- **설명:** 관리자가 수동 종료하면 읽기전용 회고 모드로 전환.
- **사용자 스토리:** "진행자로서 원하는 시점에 대화를 닫고, 전체 여정을 되돌아보고 싶다."
- **수용 기준:**
  - [ ] "세션 종료" 시 Session status=`closed`, 학생 입력이 잠긴다.
  - [ ] 캔버스가 읽기전용 회고 모드로 남아 처음부터 끝까지 스크롤 가능.
  - [ ] (Should) AI가 **비위계적 voice 요약**("어떤 차이들이 살아있었나"; 승패·우세 표현 금지)을 생성한다.
  - [ ] (Should) 관리자가 세션 데이터를 JSON으로 내보낸다.

## 화면 목록 & 화면별 구성요소
1. **랜딩/입장** — 코드 입력, 역할 선택, 닉네임.
2. **학생 무대** — 현재 질문, 응답 입력칸, 제출/대기 상태. (반응 모드 ON 시: 동등 군집 카드 열람 + 코멘트/연결.)
3. **관리자 진행 화면** — 가로 캔버스(내부 지표 포함) + 진행 패널(질문 입력·발행, 수집 마감, AI 군집화 실행, 군집/후속질문 선택·편집, 학생 모드 토글, 세션 종료) + 설정(API 키·모델).
4. **교사 모니터링 화면** — 가로 캔버스(무대 규칙) + 관찰 지표 패널(참여 현황, 분포, 폴리포니 미터). 조작 없음.
5. **회고 모드** — 잠긴 읽기전용 캔버스 + (옵션) AI voice 요약 + 내보내기 버튼.
6. **설정 모달** — Anthropic API 키, 모델 선택, 학생 반응 모드 토글.

## 데이터 모델 / 스키마 (Firestore)
```
sessions/{code}
  title:string, status:'active'|'closed', currentRoundId:string,
  studentReactionMode:boolean, model:string, createdAt:ts
sessions/{code}/rounds/{roundId}
  index:number, question:string,
  direction:'seed'|'diverge'|'converge',
  status:'waiting'|'collecting'|'closed'|'clustered'|'published',
  parentRoundId:string|null
sessions/{code}/opinions/{opinionId}
  roundId:string, participantId:string, text:string, createdAt:ts
sessions/{code}/clusters/{clusterId}
  roundId:string, label:string, summary:string,
  opinionIds:string[], parentClusterIds:string[], colorIndex:number
sessions/{code}/participants/{participantId}
  nickname:string, role:'student'|'facilitator', joinedAt:ts
sessions/{code}/reactions/{reactionId}   // 옵션
  roundId, clusterId, participantId, type:'link'|'comment', text
```
- **우세 은폐는 표현 계층 책임:** 데이터엔 `opinionIds`(=인원) 보존, 학생/교사 무대 컴포넌트는 길이/카운트를 렌더링하지 않는다.

## AI / 핵심 동작 (Anthropic Claude)
- **호출 위치:** 관리자 기기 전용. SDK 또는 fetch로 Messages API 호출, 키는 localStorage.
- **모델:** 기본 `claude-opus-4-8`, 경량 옵션 `claude-haiku-4-5`. (정확한 모델 ID·파라미터는 구현 단계에서 claude-api 스킬로 확인.)
- **프롬프트 1 — 군집화:** 입력=의견 배열. 출력(JSON)=군집 배열(label, summary, opinionIds) + outliers. **지시:** 순위·다수·우세를 매기지 말 것, 외톨이 의견을 억지로 흡수하지 말 것, 차이를 보존할 것.
- **프롬프트 2 — 후속질문 제안:** 입력=현재 군집. 출력=세분(diverge)용 질문 2~3 + 통합(converge)용 질문 2~3. **지시:** 정답 유도 금지, 서로 다른 관점을 더 벌리거나(세분) 가로지르는(통합) 질문.
- **프롬프트 3 — 회고 요약(옵션):** 입력=전체 라운드/군집. 출력=비위계적 voice 지도(승패·우세 표현 금지).
- 출력은 구조화 JSON으로 강제(파싱 안정성). 실패 시 재시도/관리자 수동 편집 폴백.

## 제안 기술 스택 & 근거
- **React + Vite (JS/JSX):** TS 미사용 요구 반영. 실시간 상태·캔버스에 적합.
- **Firebase Firestore:** 실시간 구독으로 다중 기기 동기화. 익명 인증 또는 무인증 + 보안 규칙.
- **Anthropic SDK(@anthropic-ai/sdk) 또는 fetch:** 군집화·질문제안·요약.
- **SVG + CSS:** 열 레이아웃과 분기 선 시각화(외부 무거운 차트 라이브러리 지양).
- **상태:** React Context + Firestore 구독 훅(예: `useSession`, `useRound`).

## 비기능 요구사항
- **성능:** 동시 학생 ~30–40, 발행/군집 반영 수 초 내.
- **보안:** API 키 관리자 기기 한정. Firestore 보안 규칙으로 학생의 군집·타인 응답 수정 차단. 키를 Firestore/학생 기기로 전송 금지.
- **접근성:** 한글 가독성 우선, 색에만 의존하지 않는 voice 구분(라벨 병기).

## 마일스톤 제안
- **M1 — 뼈대:** Vite+React+Firebase 셋업, 세션 생성/참가, 질문 발행→응답 수집→Firestore 실시간 반영(군집 전). 3뷰 라우팅.
- **M2 — 사이클 + AI:** 관리자 기기 Claude 군집화, 군집 확정, 후속질문 제안·선택, 라운드 연쇄. 설정(API 키).
- **M3 — 캔버스(열 뷰) + 우세 은폐:** 가로 스크롤 열 렌더링, 무대/관리자 표현 분리, 연결선.
- **M4 — 종료/회고 + 모니터링 지표:** 수동 종료, 읽기전용 회고, 참여 현황·분포.
- **M5 (Should) — 분기 뷰, 폴리포니 미터, AI 회고 요약, JSON 내보내기.**
- **M6 (Could) — 학생 반응 모드, 자동 종료, 군집 수동 편집.**

## 미해결 / 개발 전 결정 필요 사항
- **응답 정책 가정:** 1인 1응답·수정 가능으로 가정 — 라운드당 복수 응답 허용 여부 확정 필요.
- **인증 수준:** 무인증 코드 기반 가정 — 관리자 화면 보호용 간단 키를 둘지.
- **Firestore 보안 규칙 구체화:** 역할별 쓰기 권한(특히 학생 쓰기 범위) 정의 필요.
- **모델 ID/파라미터:** 구현 시 claude-api 스킬로 최신 모델·토큰·툴유스 형식 확인.
- **분기 뷰 가독성:** 의견 수가 많을 때의 축약/그룹핑 전략(Should 단계에서 결정).
- **i18n:** 현재 한국어 전용 가정.
